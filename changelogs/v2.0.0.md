# 更新日志 - v2.0.0 (互动答题机制升级)

本次更新为群聊答题带来了全新的**大模型互动判分与抢分机制**，极大地提升了群组内的答题可玩性与活跃度。

## 🌟 新增功能

### 1. 互动判分系统 (LLM Judge)
* 增设 `/a <题目ID> <你的回答>` 命令：用户可以直接向机器人提交问题的回答文字，支持答案中存在空格或者换行。
* 接入大模型判阅逻辑，大模型将根据回答是否命中考点（`score_points`）进行智能分析、计分，并给出幽默犀利的点评。
* 加入防作弊体系：如果复制粘贴其他人的答案或网上搜索内容，将被大模型识破并取消得分。
* 加入最低字数限制防止无效水频。

### 2. 积分与经验双轨制经济系统
* **抢分模式 (Score)**：题库支持设定子考点。第一个答对该考点的群友将获得对应的硬核积分（Score），考点一旦被抢走，后续回答将不再得分。直到满分被掏空。
* **低保经验 (EXP)**：为了鼓励开口，只要对题目做出有效的尝试，都能获得保底的经验值（EXP）。
* **冷却防刷机制**：全局防刷配置支持设定基础经验冷却时间（默认为 30 天，astrbot 管理员可在 AstrBot 插件面板直接修改）。即便是随机抽到的相同古老题目重复作答，若仍在冷却期，也不会反复赐予基础经验。
* **分领域权重设置**：不同技术领域允许在数据库中设定不同的 `base_exp`。

### 3. 未解题答案保护与下一考点提示
* **防剧透答案锁**：如果今日题目已被激活但是尚未被大家合力把分抢完，普通用户调用 `/ans` 偷看答案的行为将被禁止拦截。
* **顺次考点解密**：新增 `/h <题目ID>`。当群内遇到抢分瓶颈时，大模型会分析出**当前仍未被人回答出来的考点**，给出定向的侧面引导与提示，帮助大家突破瓶颈！
* **特权透视**：新增管理员指令 `/vans <题目ID> <答案字段类型>`，使管理员可以无视任何防作弊保护逻辑，随时自由审阅原始标准题解文本。

### 4. 荣耀展板系统
* **雷达图个人成绩**：`/myscore` 支持跨领域分别统计的专精展板（包含各子领域的 EXP 积累、积分以及在各该领域所处进度的直观占比）。
* **群组专项战神榜**：新增指令 `/lrank [领域名]`。不仅能直接查询总榜单本群大牛，更支持按特定领域（比如 `/lrank Java`）查询专项人才排行榜单。

## 🛠 代码与架构优化

* **功能职责重构**：打破了所有混杂命令大锅饭的局面，剥离成立了独立的 `query.py`、`answer.py`、`problem.py`、`user.py`、`admin.py` 进行松耦合注册，极大地方便了后续查错和二次开发。
* **Schema表结构补全**：全面预引了关于作答记录、组别总分等相关的聚合追踪。
* 支持了 `AstrBot` 前台面板中的可视化冷却参数选项。
* **数据安全增强**：底层全面采用 `dataclass` 进行封装，将以往的字典/Row 访问（`[]`）重构为属性访问（`.`），彻底消除了“对象不支持下标访问”类的潜在错误。
* **性能优化**：为关键日志表增加了索引，大幅提升了历史记录的查询速度。

## 🐞 修复与体验优化
* **稳定性修复**：修复了调度器初始化时部分类缺失导入导致的启动失败问题。

## ⚠️ 数据库迁移指南 (v1.x -> v2.0)

如果你是从 `v1.x.x` 升级到本版本，你需要在astrbot的根目录下执行以下python脚本来迁移数据库：

由于 v2.0 引入了多分类支持，数据库架构发生了较大变化。**直接运行新版本会导致原有群进度、积分记录不可见**。
请在更新代码后，在插件根目录下创建 migrate.py 并运行（或直接参考以下脚本逻辑）：

```python

import sqlite3
import os
import shutil
from pathlib import Path

# 领域 ID 映射逻辑 (v1.x 默认域 -> v2.0 分类 ID)
DOMAIN_MAP = {
    1: 1,   # Java -> Java基础
    3: 7,   # Golang -> Go语言基础
    4: 8,   # MySQL -> MySQL面试题
    5: 9,   # Redis -> Redis面试题
    6: 10,  # 网络协议 -> 网络协议面试题
    7: 11,  # 操作系统 -> 操作系统面试题
    9: 13,  # 消息队列 -> 消息队列面试题
    11: 15, # 系统设计 -> 系统设计面试题
}

def migrate():
    # 1. 自动定位关键文件
    plugin_dir = Path(__file__).parent.resolve()
    
    # 查找数据库路径 (按优先级)
    db_search_paths = [
        plugin_dir.parent.parent / "data" / "plugin_data" / "astrbot_plugin_group_quiz" / "quiz.db",
        plugin_dir / "quiz.db",
        Path.cwd() / "data" / "plugin_data" / "astrbot_plugin_group_quiz" / "quiz.db"
    ]
    
    db_path = next((p for p in db_search_paths if p.exists() and p.stat().st_size > 0), None)
    
    if not db_path:
        print("❌ 错误：找不到现有的 quiz.db 文件。")
        print("请手动指定路径或确保您在 AstrBot 环境中运行此脚本。")
        return

    # 备份原始文件
    backup_path = db_path.with_suffix(".db.bak")
    shutil.copy2(db_path, backup_path)
    print(f"✅ 已创建数据库备份: {backup_path}")

    conn = sqlite3.connect(db_path)
    conn.row_factory = sqlite3.Row
    cursor = conn.cursor()

    try:
        print("-> 正在读取旧数据...")
        # 备份核心进度和分数
        tables = ['users', 'subscribes', 'group_task_config', 'user_answer_log', 'problem_score_log', 'problems']
        backups = {}
        for table in tables:
            try:
                cursor.execute(f"SELECT * FROM {table}")
                backups[table] = [dict(row) for row in cursor.fetchall()]
                print(f"   已读取 {table}: {len(backups[table])} 条记录")
            except:
                backups[table] = []

        if not backups['group_task_config']:
            print("⚠️ 未发现有效的旧记录，请确认路径是否正确。")
            return

        # 2. 清理旧架构 (保留 sqlite_sequence)
        print("-> 重建数据库架构...")
        cursor.execute("PRAGMA foreign_keys = OFF;")
        cursor.execute("SELECT name FROM sqlite_master WHERE type='table' AND name NOT LIKE 'sqlite_%'")
        for t in [r[0] for r in cursor.fetchall()]:
            cursor.execute(f"DROP TABLE IF EXISTS {t}")

        # 加载新架构
        schema_file = plugin_dir / "sql" / "schema.sql"
        insert_file = plugin_dir / "sql" / "insert.sql"
        
        with open(schema_file, 'r', encoding='utf-8') as f:
            cursor.executescript(f.read())
        with open(insert_file, 'r', encoding='utf-8') as f:
            cursor.executescript(f.read())

        # 3. 恢复并映射数据
        print("-> 正在执行数据分类映射...")
        
        # 恢复用户数据 (users/subscribes)
        for u in backups['users']:
            cursor.execute("INSERT OR IGNORE INTO users VALUES (?,?)", (u['qq'], u['username']))
        for s in backups['subscribes']:
            cursor.execute("INSERT OR IGNORE INTO subscribes (user_qq, group_id) VALUES (?,?)", (s['user_qq'], s['group_id']))

        # 核心映射：处理群推送状态
        for cfg in backups['group_task_config']:
            d_id = cfg['domain_id']
            # 将旧的 domain_id 映射到新的 category_id，不存在则默认 1
            cat_id = DOMAIN_MAP.get(int(d_id), 1)
            cursor.execute("""
                INSERT INTO group_task_config (group_qq, domain_id, push_time, is_active, now_category_id, now_cursor, strategy_type)
                VALUES (?, ?, ?, ?, ?, ?, ?)
            """, (cfg['group_qq'], d_id, cfg['push_time'], cfg['is_active'], cat_id, cfg['now_cursor'], cfg.get('strategy_type', 'batch')))

        # 恢复所有答题记录和分数
        for log in backups['user_answer_log']:
            # v1 的 log 结构与 v2 一致
            cols = ', '.join(log.keys())
            placeholders = ', '.join(['?'] * len(log))
            cursor.execute(f"INSERT OR IGNORE INTO user_answer_log ({cols}) VALUES ({placeholders})", tuple(log.values()))

        for score in backups['problem_score_log']:
            cursor.execute("INSERT OR IGNORE INTO problem_score_log (problem_id, group_qq, push_date, total_score, covered_mask, is_complete) VALUES (?,?,?,?,?,?)",
                           (score['problem_id'], score['group_qq'], score['push_date'], score['total_score'], score['covered_mask'], score['is_complete']))

        conn.commit()
        print("\n🎉 迁移完美成功！原数据已根据 v2.0 架构自动归档。")
        print("💡 请重启 AstrBot 以应用所有变更。")
    except Exception as e:
        conn.rollback()
        print(f"❌ 迁移失败：{e}")
    finally:
        conn.close()

if __name__ == "__main__":
    migrate()


```
